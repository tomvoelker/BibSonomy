Aufgabe: BibTeX mit Hilfe von JabRef exportieren
------------------------------------------------------------------------------------------------------------------------
Benutzer sollen die Möglichkeit haben, einen JabRef-kompatiblen 
Export-Filter (siehe http://jabref.sourceforge.net/help/CustomExports.html) hochzuladen. Auf irgendeiner Seite werden
die BibTex-Einträge eines Benutzers anhand seines Layouts ausgegeben. Wir gehen davon aus, dass die Benutzer die Ausgabe
auf externen Seiten einbinden wollen. 

JabRef bietet aber drei Dateien pro Layout an? Warum kann der Benutzer nur eine speichern?
JabRef baut das Layout eigentlich ohne xxx.begin.layout und xxx.end.layout, da diese nur als Header
und Footer genutzt werden. Z.B. html.begin.layout(<HTML><TITLE>Das ist der Titel</TITLE><BODY>) und
html.end.layout(</BODY></HTML>). Der Hauptteil wird mit der xxx.layout erzeugt, indem über jeden BibTeX-Eintrag iteriert
und der Inhalt anhand der Feldkommandos angeordnet wird. 
Da wir davon ausgehen, dass der Benutzer die Ausgabe irgendwo einbinden will, muss er sich um Header und Footer
"extern" kümmern. Damit brauchen wir nur eine Datei zu verwalten und der Benutzer kann seine Ausgabe flexibler nutzen.

Wie wurde es umgesetzt?
-> JabRef Version 2.2beta vom 27.11.2006: build.xml wirft ein jabref.jar ab, dass in BibSonomy eingebunden wurde.
Die Standardlayouts von JabRef wurden in den Ordner jabrefExportFilter kopiert, damit wir sie anpassen können.
Benutzer können ihre Filter auf der settings.jsp uploaden und löschen (genauso wie pdf). Dafür wurde der DocumentUploadHandler
angepasst.
Die Logik steckt in helpers.export.bibtex.ExportBibtex.java:
	Singleton, dass bei der Initialisierung die Standardlayouts als jabref-Layout-Objekte in den Hauptspeicher lädt.
	Wenn die Ausgabe für einen bestimmten Benutzer zum ersten Mal aufgerufen wird, dann holen wir uns seinen Filter und
	halten ihn ebenfalls vor.
	Falls ein Benutzer seinen Filter löscht, dann wird das Layout ebenfalls entfernt (DocumentDownloadHandler ruft unloadCustomFilter()).
