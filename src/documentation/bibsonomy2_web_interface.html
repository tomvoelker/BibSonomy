    • Requirements for Web-Interface
      • Allgemein
        • KISS
        • moeglichst einfache Einarbeitung
        • einfachstes Deployment
        • Unterstuetzung fuer unser URL-Schema
        • eingebautes Test-Framework
        • Session-Support
        • File-Upload
        • unterstuetzt User-Verwaltung
        • guter Support (FAQ, Tutorial, Buecher, ...)
        • System sollte sich bewaehrt haben
      • Technologien
        • so nah wie moeglich an unseren eingesetzten Technologien
          • Java, XML
        • einfache Einbindung von AJAX / Javascript, wird schon viel verwendet!
          • wesentliche Funktionen sollten auch ohne Javascript verfuegbar sein
        • multi-threading
      • Sicherheit
        • z.B. Generierung von zufaelligen Namen fuer Felder von Input-Forms (verhindert Replay-Attacken)
        • CSRF-Attacken (cross-site request forgery)
      • Output
        • volle UTF8-Unterstuetzung
        • vorbereitet fuer Mehrsprachigkeit (i18n?)
        • nicht zu starr, z.B. bei Erzeugung von Forms
        • Layout
          • leicht veraenderbar (z.B. via Template)
          • customizable fuer jeden User
          • valides XHTML
        • Browser
          • moeglichst unabhaengig
          • back-button "macht das richtige"
      • Input
        • eingebaute Funktionen zur Verarbeitung von GET/POST-Variablen
        • Moeglichkeiten der Validierung
      • Entwicklung
        • Anbindung an Eclipse
